info Overview

What this app is

TrustFundBD is a crowdfunding site with three roles — donors, campaign creators, and admins — built around one idea: nothing moves without verification. Campaigns need admin approval to go live, donations need a verified transaction ID, and money only leaves escrow once an admin approves a withdrawal backed by proof. A trust score quantifies how much of that verification a campaign has earned.

How the project is organized

Four Python files hold all the logic. Everything else is templates and static assets.

codeapp.py
Every route: auth, campaigns, donations, proofs, withdrawals, admin approvals.
databasemodels.py
6 database tables — User, Campaign, Donation, Proof, CampaignEdit, WithdrawRequest. Points at local SQLite by default, or a Postgres schema named pyapp on Supabase when a DATABASE_URL env var is set — same code, no changes needed either way.
verifiedtrust.py
One pure function: recalculates a campaign's 0–100 trust score.
linkledger.py
The blockchain integration seam — one function to swap out later.
descriptiontemplates/
19 server-rendered HTML pages (Jinja2), Tailwind CDN for styling.
folder_zipstatic/uploads/
Campaign images and proof documents. Saved to local disk during development; automatically uploaded to a public Supabase Storage bucket instead when deployed, since Vercel's serverless functions have no persistent filesystem.
storagetrustfundbd.db
SQLite file, created automatically the first time you run the app locally. Not used at all when DATABASE_URL is set — Postgres takes over instead.
receipt_longchain_ledger.json
The stand-in "chain" — an append-only log of trust events.
cloud_uploadvercel.json / .vercelignore / .python-version
Deployment config for Vercel — keeps the build bundle small and pins the Python version. Vercel auto-detects app.py's Flask app object; no extra entry-point file is needed.

A campaign's trust score, out of 100

Recalculated every time a donation or proof gets approved, in trust.py.

+25
Creator identity verified
+25
Approved proofs (8 pts each, capped)
+25
Updated within the last 7 days
+25
Donation approval rate
All four add up to a score between 0 and 100, shown on every campaign card and the campaign's "Trust Protocol" panel.

Where blockchain plugs in

Every approval that "establishes trust" calls one function — record_event() in ledger.py. Today it appends to a local JSON file and returns a fake hash. Swapping in a real chain means rewriting the inside of that one function; nothing else in the app changes.

donation.tx_hash = record_event("donation_approved", {"amount": donation.amount, ...})

Three commands to get started

Once you're inside the trustfundbd_python folder (see above), you'll also need Python 3.10+ installed.

1
Install dependencies — pip install -r requirements.txt
2
Start the server — python app.py
3
Open http://127.0.0.1:5000 — the SQLite database is created automatically.
cloud_done
To deploy for real: push to GitHub, import the repo on Vercel, and set a DATABASE_URL (Supabase Postgres), SUPABASE_URL, SUPABASE_KEY, and SECRET_KEY as environment variables. The app switches from SQLite to Postgres and from local file uploads to Supabase Storage automatically — no code changes needed.
